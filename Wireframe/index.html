Getting to know your GIT

One
What is Git?
Git is a version control system that helps developers manage and track changes in their code over time. It facilitates collaboration and allows users to revert to previous versions, making project organization and maintenance easier. Essentially, Git documents the evolution of a project.
To find more information: https://git-scm.com/book/en/v2/Getting-Started-What-is-Git%3F
Two
Why do developers need Git?
Developers need Git to manage code changes, collaborate effectively, and revert to previous versions easily. It improves project organization and reduces conflicts, enhancing teamwork and streamlining the development process.
To find more information: https://www.youtube.com/watch?v=9lfgB0vxYco
Three
What is a branch in Git?
A Git branch is a separate development line that lets developers work on features or fixes independently. It enables parallel tasks and can be merged back into the main branch for organization and collaboration.
To find more information: https://www.w3schools.com/git/git_branch.asp
